# Contribution

The entire project is just collection of md files. You are expected to format the markdown properly. Check for typos and lint before raising a pull request.

You can help with adding more content, improving on the existing content by adding some explanation, or by fixing any errors.

Please create issues to the github repository, when you find something wrong in terms of concept or grammer or even typos.

## FAQs

### Why file names start with number

To make sure that the files are arranged in an order. The importance of the repository is not just to be used with gitbook. but instead, you can read the content within your code editor which support markdown preview.

### Why number in filenames have gap or difference instead of being in sequence

To make it easy to add any new content between two files. If a sequence is used, and you want to add any content, you will be required to modify the filenames of all the files following the number at which you need to insert a new entry.

### Do you need to update `Summary.md` after every addition of entry

Yes, if its a new entry.
